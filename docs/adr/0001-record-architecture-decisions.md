# 1. Record architecture decisions

Date: 2026-05-28

## Status

Accepted

## Context

We need to record the architectural decisions made on this project, so the
rationale behind each choice is preserved for future maintainers and is
defensible in an interview setting.

## Decision

We will use Architecture Decision Records, as described by Michael Nygard, stored
as sequentially numbered Markdown files under `docs/adr/`.

## Consequences

Decisions live next to the code, are versioned with it, and are reviewable in the
same PR as the change they describe. Each milestone that makes an architectural
choice should add a new ADR.
