# Alembic migration revisions

Revisions are placed here by `alembic revision` (or `make migration m="msg"` for
autogenerated diffs). Files are sorted by their `YYYYMMDD_<rev>_<slug>` prefix so the
directory listing reflects chronological order.

The first migration (`*_0001_initial_schema.py`) creates the `vector` extension
**before** any table that uses a `Vector` column, then creates all M1 tables. Subsequent
milestones add migrations on top of that baseline.
